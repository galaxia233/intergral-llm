## 二、Schema 结构

### 2.1 核心字段（4 个）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| question | string | ✓ | 题目内容 |
| answer | string | ✓ | 最终答案 |
| solution | string | ✓ | 解题过程 |
| tag | struct | ✓ | 元数据标签 |

### 2.2 tag 字段结构（8 个子字段）

| 子字段 | 类型 | 说明 |
|--------|------|------|
| tools_solvable | list<string> | 可解工具列表 |
| symbolic | bool | 是否纯符号可解 |
| problem_type | string | int/sum/mix |
| pure_int | bool | 是否纯积分式 |
| have_definite | bool | 是否有定积分 |
| have_indefinite | bool | 是否有不定积分 |
| is_multi | bool | 是否多变量 |
| is_divergent | bool | 是否发散 |

## 三、Python 使用示例

### 3.1 读取 Parquet 文件

```python
import pyarrow.parquet as pq

# 读取 Parquet 文件
table = pq.read_table('积分题目示例数据.parquet')

# 转为 pandas DataFrame
df = table.to_pandas()

# 查看数据
print(df.head())
```

```python

table = pq.read_table('积分题目示例数据.parquet')
```

## 四、DuckDB 使用示例

### 4.1 直接查询 Parquet

```python
import duckdb

# 直接查询 Parquet 文件
result = duckdb.query("""
    SELECT question, answer, tag.problem_type
    FROM '积分题目示例数据.parquet'
    WHERE tag.have_definite = true
""").fetchdf()

print(result)
```

### 4.2 导入 DuckDB 表

```python
from database.api import DB

with DB() as db:
    # 从 Parquet 导入
    db.from_parquet('积分题目示例数据.parquet', 'integral_problems')
    
    # 查询
    results = db.execute("SELECT COUNT(*) FROM integral_problems")
    print(f"总题数：{results[0]}")
    
    # 按类型统计
    stats = db.execute("""
        SELECT tag.problem_type, COUNT(*) as count
        FROM integral_problems
        GROUP BY tag.problem_type
    """)
    print(stats)
```

## 五、数据约束

### question 字段（7 条约束）
1. 运算符号用文本表达（不用 Unicode 数学符号）
2. 格式灵活（支持"积分式="或自然语言描述）
3. 完整自洽（题目必须包含解题所需的全部信息）
4. 符号说明完整（所有出现的符号必须说明其意义）
5. 唯一积分操作（一道题只能包含一个积分操作）
6. 非证明题格式（不能是证明题，必须是计算题）
7. 支持积分/求和（可以是积分题或求和题或混合形式）

### answer 字段（3 条约束）
1. 仅提供最终结果表达式
2. 不包含解题过程
3. 保持最简形式

### solution 字段（2 条约束）
1. 使用纯文本格式（不支持 LaTeX/Markdown）
2. 解题过程为直接从数学书籍上摘录的文本
